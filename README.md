# Namescape

*Geospatial linguistic landscape analysis from OpenStreetMap points of interest.*

A small web app that shows, for a chosen set of location types (shops,
places of worship, government offices, …) and a list of keywords, the
**fraction of locations whose name matches any keyword** — mapped per
administrative district with a configurable color palette. Currently
wired to Istanbul's 39 ilçe; the same pipeline extends to any city with
administrative boundaries in OSM.

Data comes from **OpenStreetMap** via the Overpass API. No API key, free.

## What it does

1. You pick location types with checkboxes (hierarchical: category-level
   "all", or individual OSM tags like `shop=bakery`).
2. You type keywords (one per line).
3. You pick a matching mode:
   - **substring** — diacritic-insensitive substring match (fast, loose)
   - **stemmed** — Turkish Snowball stemmer + diacritic-insensitive
     (default, handles suffixed forms: `kitap` → `kitabı`, `kitaplar`, …)
   - **regex** — your Python regex, case-insensitive
4. You click Run. The backend queries Overpass (once per unique tag set,
   cached on disk), assigns each location to its district by point-in-polygon,
   and returns per-district `{count, matches, fraction}`.
5. The map recolors districts by the fraction using the selected palette.
   Districts with zero matching locations are gray/hatched.
6. Click a district to see its matching names and a sample of all names.

## Install

Any Python 3.10+ environment works. For example, a dedicated conda env:

```
conda create -n namescape python=3.11
conda activate namescape
pip install 'fastapi>=0.110' 'uvicorn[standard]>=0.27' 'httpx>=0.27' \
            'shapely>=2.0' 'snowballstemmer>=2.2' 'osm2geojson>=0.2.4'
```

(Env name is arbitrary — whatever you already have works.)

## Run

```
conda activate namescape   # or whatever env has the deps
./run.sh
```

or

```
python -m backend.setup_districts   # one-time, ~30s; fetches 39 ilçe polygons
uvicorn backend.app:app --port 8765
```

Open http://127.0.0.1:8765 .

## Layout

```
backend/
  app.py              FastAPI: /api/tags /api/palettes /api/districts /api/query
  overpass.py         Builds + sends Overpass queries, caches per tag-set
  matching.py         substring / stemmed / regex keyword matching
  districts.py        Loads district GeoJSON, point-in-polygon via shapely STRtree
  setup_districts.py  One-time: fetch ilçe polygons -> backend/data/istanbul_districts.geojson
  tags_config.py      The tag tree shown in the sidebar (edit to add tags)
  palettes.py         Color stops per palette name
  data/
    istanbul_districts.geojson     (generated by setup_districts)
    cache/<sha>.json.gz            (Overpass responses, one per unique tag-set)
frontend/
  index.html  app.js  style.css    Leaflet map + sidebar
run.sh
```

## Notes on caching

Queries are keyed by the exact set of selected OSM tags. Changing only the
**keywords** or the **palette** does not re-query Overpass — matching runs
over the cached element list. Changing the tag selection (or any filter)
triggers a new Overpass query unless that tag set was queried before.

To force a re-fetch, delete the relevant file under `backend/data/cache/`.

## Notes on district assignment

- Polygons come from OSM `admin_level=6` relations inside Istanbul
  (relation 223474). There are typically ~39 ilçe on both European and
  Asian sides.
- For nodes we use their lat/lon; for ways/relations we use `out center`
  (Overpass centroid of the way/relation bounding geometry). Good enough
  for assigning a shop to a district.

## Adding more location types

Edit `backend/tags_config.py`. Each tag is `{label, key, value, filter_key?,
filter_value?}`. `value="*"` means "any value for this key". The frontend
rebuilds the checkbox tree from this config.
